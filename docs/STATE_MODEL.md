
# 市场状态模型（草案）

本文档用于描述「加密市场牛熊状态机」当前阶段的核心抽象思路。

⚠️ 当前状态：早期草案  
本模型尚未定型，预期会通过社区讨论与反复迭代不断演进。

---

## 一、核心思想

我们尝试用**结构化状态**，而不是情绪或单一指标，
来描述加密市场所处的宏观阶段。

市场被抽象为两个维度：

- **趋势方向**（上行 / 下行）
- **资金行为**（进攻 / 防守）

由此形成一个四象限状态模型：


```markdown
资金进攻              资金防守
      ┌─────────────┐      ┌─────────────┐
趋势多 │  牛市进攻   │      │  牛市修复    │ 
      │  🔥 HIGH    │      │  📈 MEDIUM │
      └─────────────┘      └─────────────┘
      ┌─────────────┐      ┌─────────────┐
趋势空│  熊市反弹    │      │  熊市消化    │
      │  ⚡ MEDIUM │      │  🩸 LOW     │
      └─────────────┘      └─────────────┘
```

## 二、状态的基本直觉（非严格定义）

> 以下描述是**认知直觉层**，不是最终数学定义。

### 🔥 牛市进攻
- 趋势明确向上
- 资金主动进场（风险偏好高）
- 新资金推动价格创新高

---

### 📈 牛市修复
- 大趋势仍在，但短期震荡 / 回撤
- 资金开始防守，但尚未全面撤离
- 常见于牛市中期的调整阶段

---

### ⚡ 熊市反弹
- 大趋势仍偏空
- 价格出现技术性反弹
- 资金偏交易性、非长期配置

---

### 🩸 熊市消化
- 趋势向下或横向钝化
- 资金持续流出或极度谨慎
- 市场以“时间换空间”为主

---

## 三、状态转移的核心问题（Open Questions）

以下问题**尚未有标准答案**，是当前最欢迎讨论和贡献的部分：

- 趋势应该如何被形式化定义？
  - MA？结构高低点？多周期？
- 哪些信号是：
  - 「状态定义信号」
  - 哪些只是「确认信号」？
- 当信号出现冲突时：
  - 应该加权？
  - 还是进入“中性/模糊态”？
- 状态切换是否需要：
  - 时间确认？
  - 连续 N 日条件？

---

## 四、设计原则（暂定）

- 不追求预测价格
- 不追求高频切换
- 更偏向 **中周期结构判断**
- 所有状态必须：
  - 可解释
  - 可回测
  - 可复盘
